# OrangeHRM – QA Manual Testing Project

## About the Project

This repository contains manual tests executed on the OrangeHRM demo application, initially focused on the Login module.

The purpose of this project is to demonstrate practical QA skills through structured test documentation, exploratory analysis, and professional bug reporting.

---

## Project Goals

- Practice system analysis
- Define a basic test strategy
- Identify positive and negative test scenarios
- Create structured test cases
- Execute manual tests
- Report bugs with severity classification
- Organize testing artifacts in a professional way

---

## Application Under Test

- System: OrangeHRM (Demo Version)
- URL: https://opensource-demo.orangehrmlive.com/
- Environment: Public Demo
- Browser: Google Chrome
- Operating System: Windows

---

## Testing Approach

The testing process included:

- Functional Testing
- Validation Testing
- Exploratory Testing
- Negative Testing (invalid inputs and edge cases)

All identified defects were documented using a structured bug report model containing:

- Title
- Environment
- Preconditions
- Steps to Reproduce
- Expected Result
- Actual Result
- Severity
- Evidence (Screenshots)

Severity was classified considering business impact, system functionality impact, and user experience.
